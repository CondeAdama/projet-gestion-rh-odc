1. PRÉSENTATION
---------------
Application de Gestion des Ressources Humaines (MINERVA GROUP / ODC).
Architecture modulaire Spring Boot + React, avec :
  - Autorisation granulaire par module et action (dont AFFICHER_AUTRUI)
  - Suppression logique à deux étapes (corbeille + restauration)
  - Notifications e-mail / SMS configurables depuis l'interface
  - Modèles de messages éditables (activation, congés, paie, tests…)
  - Pointage multi-passages (entrées/sorties/ré-entrées illimitées par jour)
  - Export documents (bulletins, badges) : Imprimer, PDF, Image
  - 14 fournisseurs SMS paramétrables (Twilio, Brevo, Termii, HTTP…)

Stack : Spring Boot 3.5 + React 19 + Vite + MariaDB/MySQL

Local (développement) :
  API   : http://localhost:8080/api/v2
  Web   : http://localhost:5173

Production (cloud — gratuit, validé le 2026-07-07) :
  Frontend Netlify : https://minerva-rh.netlify.app          ← principal
  Frontend Render  : https://minerva-rh.onrender.com         ← secondaire
  Backend Render   : https://minerva-rh-api.onrender.com/api/v2
  Base Aiven MySQL : minerva-mysql-minerva-group.h.aivencloud.com:23766
  GitHub           : https://github.com/CondeAdama/projet-gestion-rh-odc

5. SYSTÈME D'AUTORISATION
-------------------------
Modèle granulaire : permissions par module × action.

Modules :
  EMPLOYES, CONTRATS, CONGES, PRESENCES, PAIES, RAPPORTS,
  UTILISATEURS, ROLES, REFERENTIELS, VISITES, CONFIGURATION

Actions :
  AFFICHER, AJOUTER, MODIFIER, SUPPRIMER, AFFICHER_AUTRUI

AFFICHER_AUTRUI : voir les données des autres employés (bulletins, congés…).
Sans cette permission, un employé ne voit que ses propres données.

Rôles pré-configurés :
  - ADMINISTRATEUR : accès total (badge « Admin » dans la sidebar)
  - RH             : gestion RH complète + AFFICHER_AUTRUI
  - EMPLOYE        : self-service (ses congés, ses paies, son profil)
  - RECEPTION      : pointage + visites + PAIES:AFFICHER + CONGES:AFFICHER/AJOUTER

Rôle par défaut :
  - Champ parDefaut sur Role — configurable dans Gestion Rôles
  - GET  /api/v2/roles/par-defaut
  - PUT  /api/v2/roles/{id}/par-defaut

Attribution ADMINISTRATEUR :
  - Uniquement via Gestion Utilisateurs → Modifier (admin connecté uniquement)
  - Interdit à la création d'employé

8. MODULE PRÉSENCES (multi-passages)
------------------------------------
  POST /api/v2/presences/scanner?matricule=&localisationId=
  GET  /api/v2/presences/aujourdhui
  GET  /api/v2/presences/employe/{id}
  GET  /api/v2/presences/recherche        → filtres date, site, département, statut
  GET  /api/v2/presences/statistiques     → passages, employés, en règle, retards, sur site
  GET  /api/v2/presences/corbeille
  DELETE /api/v2/presences/{id}
  POST   /api/v2/presences/{id}/restaurer

Logique scan :
  - Passage ouvert (sans sortie) → enregistre la SORTIE
  - Sinon → nouvelle ENTREE (passage #N)
  - Retard (après 08:30) uniquement sur la 1ère entrée du jour
  - Plusieurs passages par employé et par jour autorisés

Frontend :
  /dashboard/pointage  → PointageQR (scan, filtres, stats réactives aux filtres)
  /scan                → PointageMobile (plein écran, caméra)
